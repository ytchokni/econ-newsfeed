# Product Guidelines: econ-newsfeed

## Voice and Tone

**Concise and direct.** Documentation and UI text should be clear, minimal, and to the point. Avoid unnecessary verbosity.

## Design Principles

1. **Simplicity over features** — Build only what is needed. Resist scope creep. A smaller, well-executed feature set beats a bloated one.
2. **Performance first** — Optimize for speed and efficiency. Minimize API calls, reduce page load times, and keep the scraping pipeline lean.

## UI Guidelines

- Prioritize readability and scannability in the newsfeed
- Use clear typography and whitespace
- Minimal chrome — let the content (publications) be the focus
- Mobile-friendly but desktop-primary

## Documentation Standards

- Keep READMEs and docs short and actionable
- Prefer examples over lengthy explanations
- Document the "why" more than the "what"
