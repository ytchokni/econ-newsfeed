# Product Definition: econ-newsfeed

## Project Name

econ-newsfeed

## Description

A system that tracks economists' personal websites, detects new publications, and displays them in a chronological newsfeed.

## Problem Statement

Economists publish new working papers and journal articles on their personal websites, but there is no unified feed to track these updates across researchers — manually checking dozens of websites is impractical.

## Target Users

Economics researchers and PhD students who want to follow new publications from specific economists.

## Key Goals

1. **Automate publication discovery** — eliminate the need for manual website checking
2. **Reliable, low-overhead scraping pipeline** — dependable background processing with minimal operational burden
3. **User-friendly newsfeed UI** — clean, chronological feed that surfaces new publications effortlessly

## Scope (MVP)

**In scope:**
- Newsfeed website showing publications sorted by discovery date
- REST API serving publication and researcher data
- Automated scraping on a configurable schedule
- Researcher directory page
- Manual scrape trigger via API
- Local development setup
- Docker Compose deployment

**Out of scope (post-MVP):**
- User accounts and authentication
- Email/RSS notifications
- Full-text search
- Publication deduplication across researchers
- Admin dashboard for managing researchers
- Custom scraping rules per website
