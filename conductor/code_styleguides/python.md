## Type Hints

- Use built-in generics (`list[str]`, `dict[str, int]`) over `typing` equivalents
- Use `|` for union types: `str | None` instead of `Optional[str]`
- Annotate return types on all public functions

```python
def get_publications(page: int = 1, per_page: int = 20) -> dict[str, Any]:
    ...
```

## Error Handling

- Use specific exception types, not bare `except:`
- Validate at system boundaries (API input, LLM output, external HTTP responses)
- Use Pydantic models for structured validation

## Docstrings

- Use docstrings only for non-obvious public functions
