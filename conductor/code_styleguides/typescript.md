# TypeScript Style Guide: econ-newsfeed

## General

- Target TypeScript 5.x with strict mode
- Follow Next.js conventions for file and component organization
- Use ESLint with `next/core-web-vitals` and `next/typescript` extends (existing config)

## Types

- Prefer `interface` for object shapes, `type` for unions/intersections
- Export types from a shared location when used across multiple files
- No `any` — use `unknown` and narrow with type guards

```typescript
interface Publication {
  id: number;
  title: string;
  authors: Author[];
  year: string;
  venue: string;
  sourceUrl: string;
  discoveredAt: string;
}

interface PaginatedResponse<T> {
  items: T[];
  total: number;
  page: number;
  perPage: number;
  pages: number;
}
```

## Data Fetching

- Use SWR for client-side fetching with stale-while-revalidate caching
- Centralize API calls in `lib/api.ts`
- Use environment variable `NEXT_PUBLIC_API_URL` for API base URL

```typescript
// lib/api.ts
const API_BASE_URL = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000";

export async function getPublications(page = 1, perPage = 20): Promise<PaginatedResponse<Publication>> {
  const res = await fetch(`${API_BASE_URL}/api/publications?page=${page}&per_page=${perPage}`);
  if (!res.ok) throw new Error("Failed to fetch publications");
  return res.json();
}
```

## Styling

- Use Tailwind CSS utility classes
- Avoid inline styles
- Extract repeated patterns into component-level abstractions only when used 3+ times

## Testing

- Write tests before implementation (TDD — strict)
- Use Jest/Vitest with React Testing Library for component tests
- Test files co-located: `PublicationCard.test.tsx` alongside `PublicationCard.tsx`
